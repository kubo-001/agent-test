# 测试用例生成示例

## 示例场景

**需求**: 为用户管理模块生成测试用例

## 完整执行流程

### Step 1: Lead分析需求

```markdown
Lead读取需求文档，分析并提取功能点：

需求文档内容：
- 用户注册（用户名、密码、邮箱验证）
- 用户登录（用户名/邮箱、密码错误锁定）
- 用户信息修改（昵称、邮箱、密码）
- 用户注销（Token验证、数据删除、冷却期）

Lead输出 features.md:

# 功能点清单

| ID | 功能点 | 描述 | 优先级 |
|----|--------|------|--------|
| F1 | 用户注册-用户名验证 | 6-20字符，字母数字下划线 | P0 |
| F2 | 用户注册-密码验证 | 8-32字符，大小写+数字 | P0 |
| F3 | 用户注册-邮箱验证 | 标准邮箱格式 | P0 |
| F4 | 用户登录-账号密码 | 用户名/邮箱+密码 | P0 |
| F5 | 用户登录-错误锁定 | 3次错误锁定15分钟 | P0 |
| F6 | 信息修改-昵称 | 修改昵称 | P1 |
| F7 | 注销-Token验证 | 注销需登录状态 | P0 |
| F8 | 注销-数据删除 | 注销后删除数据 | P0 |
| F9 | 注销-冷却期 | 1小时不可重新注册 | P1 |

任务分配：
- Generator-1: [F1, F2, F3, F4, F5]
- Generator-2: [F6, F7, F8, F9]
```

### Step 2: Generator并行编写

#### Generator-1 输出示例

```markdown
# 测试用例集 for 用户注册+登录模块

### TC-001: 用户名过短（边界）
- **前置条件**: 无
- **测试步骤**: 1. 输入用户名"abc" 2. 输入合规密码和邮箱 3. 点击注册
- **预期结果**: 注册失败，提示"用户名长度需6-20字符"
- **优先级**: P0
- **测试类型**: 边界

### TC-002: 用户名过长（边界）
- **前置条件**: 无
- **测试步骤**: 1. 输入27字符用户名 2. 输入合规密码和邮箱 3. 点击注册
- **预期结果**: 注册失败，提示"用户名长度需6-20字符"
- **优先级**: P0
- **测试类型**: 边界

### TC-003: 注册成功-正常流程（功能）
- **前置条件**: 无
- **测试步骤**: 1. 输入合规用户名、密码、邮箱 2. 点击注册
- **预期结果**: 注册成功，返回用户ID
- **优先级**: P0
- **测试类型**: 功能

### TC-004: 登录-密码错误3次锁定（功能）
- **前置条件**: 用户已注册
- **测试步骤**: 1. 输入正确用户名 2. 连续输入3次错误密码
- **预期结果**: 账户锁定15分钟
- **优先级**: P0
- **测试类型**: 功能

...（共25个用例）
```

#### Generator-2 输出示例

```markdown
# 测试用例集 for 信息修改+注销模块

### TC-101: 修改昵称成功（功能）
- **前置条件**: 用户已登录
- **测试步骤**: 1. 进入个人资料 2. 输入新昵称 3. 保存
- **预期结果**: 修改成功，显示新昵称
- **优先级**: P0
- **测试类型**: 功能

### TC-102: 注销-Token缺失（异常）
- **前置条件**: 未登录
- **测试步骤**: 1. 直接访问注销页面
- **预期结果**: 跳转到登录页
- **优先级**: P0
- **测试类型**: 异常

...（共15个用例）
```

## 如何在其他AI系统中复用

### 1. 复制文件

```
复制以下目录到目标系统：
- skills/test-generation/     → 目标skills/
- rules/test-generation-workflow.md → 目标rules/
```

### 2. 修改配置

根据目标系统调整：
- 工作目录路径
- 输出目录结构
- Agent调用参数

### 3. 触发使用

```markdown
用户: 生成测试用例
AI: 自动读取skill和rule，执行Lead→Generator→Reviewer流程
```
